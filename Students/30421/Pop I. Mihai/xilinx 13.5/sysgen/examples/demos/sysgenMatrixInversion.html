Matrix inversion using Squared Givens Rotations

Matrix inversion is used in a variety of scenarious including adaptiove estimation, beamforming etc. This design implements matrix inversion using floating point capability in SysGen. Matrix inversion is realized by first triangularizing the input matrix and then computing the inverse using backsubstition. Computation of matrix inversion can therefore, be posed as follows:

A*X = I - (i)

T*A*X = TI -(ii)

Here we determine T such that T*A becomes an upper triangular matrix(U). The above equation therefore simplifies to :

UX = TI - (iii)

Using (iii) it is possible to compute X using back substitution. This method of computing the inverse is generally more stable than using the direct adjacent matrix approach. Many algorithms exist to find T including Givens Rotations. In this example we have adopted Squared Givens Rotations, a variation of Givens Rotation as described by R. Dohler, to perform triangularization of matrix A.

The model provided in this design consists of two Subsystems. SquaredGivensRotations performs triangularization of the input matrix using the algorithm described in Reference 2.(See below). This particular subsystem is programmatically generated using sysgenMatrixInversSGRGeneration script provided with this design.
