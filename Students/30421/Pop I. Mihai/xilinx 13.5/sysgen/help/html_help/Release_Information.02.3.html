Release Notes 12.3
AXI4 Support in System Generator
AMBA® AXI™4 (Advanced eXtensible Interface 4) is the fourth generation of the AMBA interface defined and controlled by ARM®, and has been adopted by Xilinx as the next-generation interconnect for FPGA designs. The AMBA AXI4 interface connections are point-to-point and come in three different flavors: AXI4, AXI4-Lite and AXI4-Stream.
•
AXI4 is a memory-mapped interface which support burst transactions
•
AXI4-Lite is a lightweight version of AXI4 and has a non-bursting interface
•
AXI4-Stream is a high-performance streaming interface for unidirectional data transfers (from master to slave) with reduced signaling requirements (compared to AXI4). AXI4-Stream supports multiple channels of data on the same set of wires.
Refer to the topic AXI Interface for more detailed information on the AXI Interface.
Four New AXI-Stream Blocks
As shown in the figure that follows, there are four new DSP blocks that support the AXI-Stream interface. The blocks include Complex Multiplier 4.0, DDS Compiler 5.0, FIR Compiler 6.0 and Fast Fourier Transform 8.0.
These blocks can be found in a new Library named AXI4.
New AXI-Stream Block Appearance
As shown in the figure below, the ports of the same channel on the new AXI-Stream blocks are grouped together and color matched.
Other features include the following:
•
Multi-channel information is broken out into separate sub-ports. For example, the real and imaginary parts of a complex number are broken out
•
Port signals that are the same color as the block’s background identify non-AXI ports
•
AXI port names are shortened by default to improve readability in the Simulink diagram. You can, however, expanded the AXI names to their full length on the block. For example, the signal data_tready expands to m_axis_data_tready (m means “master” and axis means “AXI-Streaming.”
AXI4 (memory-mapped) Support Through the EDK Processor Block
AXI4 (memory-mapped) support in System Generator is available through the EDK Processor block found in the System Generator block set. The EDK Processor block allows you to connect hardware circuits created in System Generator to a Xilinx Microblaze processor; you have the option to connect to the processor via a PLB v4.6 or AXI4 interface.
If you are importing an EDK design into System Generator, the bus type (PLB v4.6 or AXI4) will be automatically detected by System Generator.
System Generator Performance Improvements
Compared to Release 12.2, System Generator for DSP has a 3x improvement for large designs in each of four different areas:
•
Compile (Ctrl-D)
•
Initialization
•
Simulation
•
Netlisting
This improvement translates to a 2-3x decrease in time for the entire flow from design loading to netlisting.
