Compilation
The Xilinx XtremeDSP Compilation block provides System Generator with useful information about how the model should be implemented for hardware co-simulation.  This information includes device information and what tools should be run to produce an FPGA configuration file.  The XtremeDSP Compilation block should be placed in the subsystem you would like to co-simulate.  This subsystem must also contain a System Generator token.  Only one compilation block is allowed per subsystem.  The XtremeDSP Compilation block overrides the Product Family, Device, Speed and Package fields of the System Generator GUI.  

When you are ready to compile a model for co-simulation, open the GUI for the System Generator token that is in the same subsystem you would like to co-simulate.  The model is automatically prepared for co-simulation when you press then generate button.  For more information, please consult the hardware co-simulation section of the System Generator reference guide.
